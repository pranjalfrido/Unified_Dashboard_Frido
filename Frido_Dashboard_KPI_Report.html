2.Key Computed Columns

These two columns are the foundational revenue metrics computed inside the unified CTE. All downstream KPIs are derived from them.

2.1 Gross Revenue — SellingPrice_Inc_GST
Channel Source Field / Expression Notes
Shopify final_total_incl_tax Direct from source — already includes GST
Amazon Seller Central item_price Direct from source — MRP-level price including tax
Flipkart price_after_discount_price_before_discount_total_discountx Direct from source — Flipkart invoice price post-discount
Myntra final_amount Direct from source
CRED / Firstcry / Pharmeasy SellingPrice Direct from Unicommerce source
Blinkit Selling_Price (exc GST) × qty × (1 + GST_Tax_Type_Code / 100) Selling_Price from price master; GST % from item SKU master
Instamart Selling_Price (exc GST) × units_sold × (1 + GST_Tax_Type_Code / 100) Selling_Price from price master; GST % from item SKU master
Zepto gross_selling_value × (1 + GST_Tax_Type_Code / 100) gross_selling_value = selling_price × sales_qty_units (already totalled in source)
Amazon Vendor Central ordered_revenue × (1 + GST_Tax_Type_Code / 100) ordered_revenue is exc-GST in VC source; GST uplift applied via item master
2.2 Net Revenue — SellingPrice_Exc_GST
Channel Source Field / Expression Notes
Shopify selling_price_excl_shipping_tax Direct from source
Amazon Seller Central item_price − item_tax Actual tax deducted per line item
Flipkart taxable_value_final_invoice_amount_taxesx Direct from source
Myntra seller_price Direct from source
CRED / Firstcry / Pharmeasy Subtotal Direct from Unicommerce source
Blinkit Selling_Price (exc GST) × qty From price master — already excl. GST per unit
Instamart Selling_Price (exc GST) × units_sold From price master — already excl. GST per unit
Zepto gross_selling_value Direct from source — already qty × unit selling price (exc GST)
Amazon Vendor Central ordered_revenue Direct from VC source — already exc-GST
Fallback (other channels) SellingPrice_Inc_GST / (1 + GST% / 100) If GST% known from item master; else divide by 1.18 (assumed 18%)
GST Rate Source: The GST_Tax_Type_Code column in Frido_Item_Master__frido_item_sku_master (dataset: sharepoint_to_gcp) is joined to the unified CTE via the master SKU code. This is the single source of truth for GST % across all channels.

3.Global Filters

All filters are applied as WHERE clauses to every query across all dashboard tabs. Filters are composable — multiple filters are AND-ed together.

Filter WHERE Clause Expression Notes
Date Range OrderDate BETWEEN start AND end Applied to all queries. Drives previous period window too.
Category COALESCE(im.Category, 'Frido') = value Joined from Frido_Item_Master__frido_item_sku_master
Sub-category COALESCE(im.SubCategory, 'Frido') = value Joined from item master
SKU ChannelSKUCode LIKE '%value%' Partial match across all channel SKUs
Sub-channel SubChannel = value E.g., MyFrido / Mobility / B2B / FBF / FBA / MFN
Voucher Code TRIM(voucher_code) IN (list) Multi-select list; TRIM handles whitespace
State UPPER(TRIM(State)) = value Normalised uppercase match
Region COALESCE(pm.Region, cm.Region) = value pm = pincode master; cm = city master fallback
City COALESCE(pm.City_L1, cm.City_L1, u.City) = value Three-level fallback across pincode master, city master, and Unicommerce city field
City Tier COALESCE(pm.City_Tier, cm.City_Tier) = value Tier 1 / 2 / 3 classification from pincode or city master
4.Previous Period Comparison Logic

Every KPI card that shows a % change badge uses the immediately preceding period of equal length.

Example: Selected range = Jun 14–Jun 20 (7 days) → Previous period = Jun 7–Jun 13 (7 days).
Formula: (Current Value − Previous Value) ÷ Previous Value × 100

Badge colours: Green = Positive change Red = Negative change
MoM and YoY comparisons (available on the All Channels tab) use the same-length window shifted by 1 month or 1 year respectively, rather than an equal-days window.

9.Blinkit Tab
Revenue computation: Blinkit does not carry selling price in the raw transaction table. Revenue is computed by joining Frido_Item_Master__productid_sku_mapping WHERE channelname = 'Blinkit', using the Selling_Price column (exc GST per unit). GST uplift uses GST_Tax_Type_Code from Frido_Item_Master__frido_item_sku_master.
9.1 KPIs
KPI Formula Query / Source
Gross Revenue (Inc GST) SUM(Selling_Price_exc × qty × (1 + GST% / 100)) blTotals — SUM(SellingPrice_Inc_GST) WHERE Channel='Blinkit'
Net Revenue (Exc GST) SUM(Selling_Price_exc × qty) blTotals — SUM(SellingPrice_Exc_GST)
Units SUM(qty_sold) blTotals — from partnerbizz_reports_v2.sales
SKUs COUNT(DISTINCT item_id) blTotals
Cities COUNT(DISTINCT city_name) blTotals

Daily Avg Revenue Gross ÷ nDays Calculated
% Change vs Prev Period (Current − Prev) ÷ Prev × 100 prevBl query

10.Instamart Tab
Revenue computation: Like Blinkit, Instamart revenue is computed from the price master. Join Frido_Item_Master__productid_sku_mapping WHERE channelname = 'Instamart', column Selling_Price (exc GST per unit). GST uplift via GST_Tax_Type_Code from item SKU master.
10.1 KPIs
KPI Formula Query / Source
Gross Revenue (Inc GST) SUM(Selling_Price_exc × units_sold × (1 + GST% / 100)) inTotals — SUM(SellingPrice_Inc_GST) WHERE Channel='Instamart'
Net Revenue (Exc GST) SUM(Selling_Price_exc × units_sold) inTotals — SUM(SellingPrice_Exc_GST)
Units SUM(units_sold) inTotals — from fact_instamart_sales_report
ASP (Avg Selling Price) Net Revenue ÷ Units Calculated
Daily Avg Revenue Gross ÷ nDays Calculated
% Change vs Prev Period (Current − Prev) ÷ Prev × 100 prevIn query
Source table: fact_instamart_sales_report (dataset: production). Units column: units_sold. No order-level data available — ASP is the primary per-unit metric.
11.Zepto Tab
Revenue computation: gross_selling_value in zepto_sales_report = selling_price × sales_qty_units — already a totalled field (exc GST). Gross Revenue is computed by multiplying by (1 + GST_Tax_Type_Code / 100) from the item master.
11.1 KPIs
KPI Formula Query / Source
Gross Revenue (Inc GST) SUM(gross_selling_value × (1 + GST% / 100)) zpTotals — SUM(SellingPrice_Inc_GST) WHERE Channel='Zepto'
Net Revenue (Exc GST) SUM(gross_selling_value) zpTotals — direct from source
Units SUM(sales_qty_units) zpTotals — zepto_sales_report
Orders SUM(orders) column from zepto_sales_report directly zpTotals subquery
ASP (Avg Selling Price) Net Revenue ÷ Units Calculated
Daily Avg Revenue Gross ÷ nDays Calculated
% Change vs Prev Period (Current − Prev) ÷ Prev × 100 prevZp query

Source table: Unicommerce_Sale_Orders_Report (dataset: production). Filtered by ChannelName LIKE 'cred%' (covers CRED, Firstcry, Pharmeasy sub-channels). Cancelled orders are excluded from revenue KPIs.
12.1 KPIs

Gross Revenue SUM(SellingPrice) from Unicommerce crTotals WHERE Channel='CRED'
Net Revenue SUM(Subtotal) from Unicommerce crTotals
Orders COUNT(DISTINCT DisplayOrderCode) crTotals

Daily Avg Revenue Gross ÷ nDays Calculated
% Change vs Prev Period (Current − Prev) ÷ Prev × 100 prevCr query

Each row in Unicommerce_Sale_Orders_Report represents one line item. DisplayOrderCode is the order-level identifier. SellingPrice is the gross (inc-GST) amount; Subtotal is net (exc-GST).
13.Myntra Tab
13.1 KPIs
KPI Formula Query / Source
Gross Revenue SUM(final_amount) mnTotals WHERE Channel='Myntra'

Orders COUNT(DISTINCT store_order_id) mnTotals — fact_myntra_orders_report
